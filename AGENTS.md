# Composition Support — Agent Reference

**Composition Helper** is a React Native / Expo music theory app (iOS, Android, Web).
Core feature: a Circle of Fifths roulette wheel that picks a random musical key and displays its full harmonic context (scale, diatonic chords, modes, cadences by genre). Freemium model via Supabase Auth + Stripe.

Monorepo root: `composition-support/` | Package manager: npm workspaces + Turborepo | Node >= 20

## State Management (Zustand)

### `useAppStore` — `packages/store/src/useAppStore.ts`

Persisted to AsyncStorage key `'composition-support-storage'` (only `currentKey`).

| State | Type | Description |
|---|---|---|
| `currentKey` | `KeyConfig` | Active musical key (`{ root, tonality }`) |
| `harmonyResult` | `HarmonyResult` | Derived from `currentKey`; recalculated on every `setKey()` |
| `isSpinning` | `boolean` | Wheel animation in progress |
| `targetKey` | `KeyConfig \| null` | Destination key set by `spin()` |

Actions: `setKey(root, tonality)`, `spin()`, `setSpinning(bool)`, `clearTarget()`

### `useUserStore` — `packages/store/src/useUserStore.ts`

Persisted to AsyncStorage key `'user-storage'` (`profile` + `billing`).

| State | Type | Description |
|---|---|---|
| `profile` | `UserProfile \| null` | Logged-in user (id, name, email, tier, memberSince) |
| `billing` | `{ method?, history }` | Payment method + transaction history |

Actions: `setProfile(profile)`, `updateUser(data)`, `clearUser()`

## Music Engine (`packages/music-engine/src/`)

### `engine.ts` — Exported functions

| Function | Signature | Purpose |
|---|---|---|
| `calculateHarmony` | `(root, tonality) → HarmonyResult` | Main API — returns full harmony context (scale + chords + cadences) |
| `getScale` | `(root, tonality) → Note[]` | Ordered notes for the given key |
| `getDiatonicChords` | `(scale, tonality) → Chord[]` | 7 diatonic chords with 7ths |
| `getCadences` | `(chords) → Record<string, Chord[][]>` | Maps `CADENCES` index arrays to actual `Chord` objects |
| `getRandomKey` | `() → KeyConfig` | Random key from CIRCLE_OF_FIFTHS × Major\|Minor |

### `constants.ts` — Key data

| Constant | Description |
|---|---|
| `CIRCLE_OF_FIFTHS` | `['C','G','D','A','E','B','F#','Db','Ab','Eb','Bb','F']` |
| `INTERVALS` | Semitone arrays for all 13 tonalities |
| `CADENCES` | Genre → chord index arrays (POP, JAZZ, CLASSICAL, REGGAE) |
| `MODE_QUALITIES` | Chord quality tables for all 7 Greek modes |
| `MODE_CHARACTERISTIC_DEGREES` | Which scale degrees are characteristic per mode |

### `types.ts` — Core types

| Type | Values / Shape |
|---|---|
| `Note` | `'C' \| 'C#' \| 'Db' \| … \| 'B'` (17 values) |
| `Tonality` | `'Major' \| 'Minor' \| 'Ionian' \| 'Dorian' \| … \| 'MajorPentatonic' \| 'MinorPentatonic'` |
| `KeyConfig` | `{ root: Note, tonality: Tonality }` |
| `Chord` | `{ name, root, quality, qualityType, chordClass, notes, degree, isCharacteristic? }` |
| `HarmonyResult` | `{ key: KeyConfig, scale: Note[], chords: Chord[], cadences: Record<string, Chord[][]> }` |

## Conventions

- **No package build step in dev**: packages expose `./src/index.ts` directly; Metro and Babel resolve them via `@cs/*` alias. Never add a build step to packages unless explicitly asked.
- **`harmonyResult` is always derived**: never store it independently — it is recalculated in `setKey()` and on store rehydration.
- **Freemium gate belongs in the UI layer**: use `useIsPremium()` hook in components; real enforcement is on the backend (RLS + Stripe webhook). Never bypass or weaken the gate.
- **Dark theme only**: only dark theme tokens exist. Do not introduce light mode.
- **Strict TypeScript**: all packages inherit `tsconfig.base.json` with `"strict": true`. No `any`, no type assertions without justification.
- **i18n is mandatory**: every user-visible string must go through `useTranslation()` from `react-i18next`. Never hardcode UI text.
- **Package naming**: `@cs/<name>` (cs = composition-support).
- **Surgical edits**: touch only the files required for the task. Do not refactor unrelated code.
